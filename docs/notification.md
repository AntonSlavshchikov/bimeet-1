# Фича уведомлений (Notifications)

## Цель
Добавить систему уведомлений, позволяющую пользователю:
- Видеть новые уведомления через иконку колокольчика с бейджем
- Просматривать список уведомлений
- Отмечать уведомления как прочитанные (по одному и все сразу)
- Удалять уведомления (по одному и все сразу)

---

## Основные сценарии

### 1. Отображение уведомлений
- В верхней панели отображается иконка колокольчика
- Рядом отображается бейдж с количеством непрочитанных уведомлений
- При клике открывается dropdown / панель со списком уведомлений

---

### 2. Список уведомлений
Каждое уведомление содержит:
- Заголовок (опционально)
- Текст
- Время получения
- Статус (прочитано / не прочитано)

Дополнительно:
- Непрочитанные визуально выделены (например, жирный текст или фон)
- Ограничение на количество отображаемых уведомлений (например, последние 20–50)

---

### 3. Действия пользователя

#### На уровне одного уведомления:
- Отметить как прочитанное
- Удалить

#### На уровне списка:
- "Отметить все как прочитанные"
- "Удалить все"

---

## Backend

### Логика

#### 1. Создание уведомлений
- Генерация при событиях системы (например: действия пользователей, системные события)
- Возможность асинхронного создания (через очередь)

#### 2. Получение уведомлений
- Сортировка по дате (новые сверху)
- Пагинация / лимит
- Фильтрация:
  - Все
  - Только непрочитанные

#### 3. Обновление статуса
- Обновление статуса прочтения:
  - по одному
  - массово (bulk update)

#### 4. Удаление
- Soft delete (предпочтительно) или hard delete
- Поддержка:
  - удаления одного уведомления
  - массового удаления

#### 5. Счетчик непрочитанных
- Быстрое получение количества (важно для бейджа)
- Возможные подходы:
  - отдельное поле/кеш
  - агрегация с индексом

---

### Нефункциональные требования
- Производительность: быстрый доступ к счетчику
- Масштабируемость: поддержка большого количества уведомлений
- Возможность расширения (типы уведомлений, приоритеты)

---

## Frontend

### 1. Колокольчик (Header)

#### Поведение:
- Иконка всегда доступна
- Бейдж:
  - показывается только если есть непрочитанные
  - максимум: `99+`

#### Состояния:
- Нет уведомлений
- Есть непрочитанные
- Загрузка

---

### 2. Dropdown / Панель уведомлений

#### Компоненты:
- Header:
  - "Уведомления"
  - кнопка "Отметить все как прочитанные"
  - кнопка "Удалить все"

- Список:
  - список уведомлений
  - скролл

- Пустое состояние:
  - "У вас нет уведомлений"

---

### 3. Элемент уведомления

#### UI:
- Заголовок / текст
- Время
- Индикатор "непрочитано"

#### Действия:
- Клик → (опционально) переход + mark as read
- Кнопка "прочитано"
- Кнопка "удалить"

---

### 4. Состояния и UX

- Оптимистичные обновления:
  - при клике "прочитано" сразу меняется UI
- Лоадеры:
  - при первой загрузке
- Empty state:
  - отсутствие уведомлений
- Ошибки:
  - retry / fallback

---

## Синхронизация

### Обновление данных:
- При открытии dropdown — подгрузка уведомлений
- Периодический polling или WebSocket (опционально)

### Обновление бейджа:
- После любого действия:
  - mark as read
  - delete
  - новые уведомления

---

## Расширения (в будущем)

- Группировка уведомлений
- Push-уведомления
- Email / mobile нотификации
- Фильтры и категории
- Deep links

---

## Критерии готовности (Definition of Done)

- Колокольчик отображается корректно
- Бейдж показывает точное количество непрочитанных
- Список уведомлений загружается и отображается
- Можно:
  - отметить одно уведомление как прочитанное
  - отметить все как прочитанные
  - удалить одно
  - удалить все
- UI корректно обновляется без перезагрузки
- Обработаны edge cases (пустой список, ошибки)
