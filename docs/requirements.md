# Приложение для организации встреч

## Технологический стек

- Backend: Go  
- Frontend: React  
- UI библиотека: Chakra UI  
- Data fetching / state management: TanStack (React Query, Table и др.)

---

# Описание приложения

Приложение предназначено для организации встреч и совместных мероприятий.  
Пользователи могут создавать события, приглашать участников, управлять расходами, проводить опросы и координировать подготовку к мероприятию.

Все взаимодействие происходит внутри обычного веб-приложения или мобильного приложения без использования Telegram-ботов или внешних чатов.

---

# User Story: Организация дня рождения

Андрей решил отпраздновать свой день рождения на коттедже вместе с друзьями. Для удобной организации он воспользовался приложением для планирования встреч.

---

# Шаг 1: Создание встречи

Андрей открывает приложение и нажимает кнопку **Создать встречу**.

Он заполняет основную информацию о событии:

- Название: День рождения Андрея  
- Дата: 15 марта 2026  
- Время: 14:00 – 22:00  
- Место: Коттедж "Лесная поляна"  
- Описание: Празднуем на природе. Будет мангал, игры и музыка.

После сохранения создается карточка события и страница встречи.

---

# Шаг 2: Добавление участников

Андрей добавляет участников в событие.

Он выбирает **14 друзей из списка пользователей приложения** или отправляет им приглашение по ссылке.

После добавления участники получают уведомление в приложении и могут:

- подтвердить участие
- отказаться
- посмотреть детали мероприятия

На странице встречи отображается список всех приглашенных.

---

# Шаг 3: Организация расходов

Андрей добавляет статьи расходов для мероприятия.

| Статья | Сумма |
|------|------|
| Аренда коттеджа | 12000 ₽ |
| Продукты для мангала | 4500 ₽ |
| Напитки | 3000 ₽ |
| Торт | 2500 ₽ |

Приложение автоматически:

1. считает общую сумму расходов
2. делит сумму на всех участников

Общая сумма: **22000 ₽**  
Количество участников: **15**

С каждого: **1467 ₽**

На странице события появляется блок со сводкой расходов:

- Общая сумма: 22000 ₽  
- С каждого: 1467 ₽  

Статьи расходов:

- Коттедж — 12000 ₽  
- Продукты — 4500 ₽  
- Напитки — 3000 ₽  
- Торт — 2500 ₽  

Каждый участник может отметить, что он **уже оплатил свою часть**.

# Шаг 5: Управление изменениями

Если организатор изменяет параметры события (например, время начала), приложение автоматически уведомляет всех участников.

Пример уведомления:

Изменение в мероприятии.

Время начала изменено:  
15:00 (было 14:00)

Все остальные детали остаются прежними.

История изменений сохраняется в событии.

# Требования: Публичные и персональные встречи

## Общее описание

В приложении необходимо реализовать два типа встреч:

- **Публичные встречи** — доступны всем пользователям и отображаются в общем списке
- **Персональные встречи** — доступны только по приглашению и скрыты от остальных пользователей

Дополнительно для публичных встреч вводится понятие:
- **Количество гостей** — ограничение на число участников

---

## Типы встреч

### Публичная встреча

Публичная встреча доступна всем пользователям приложения.

Особенности:
- отображается в общем списке встреч
- доступна для просмотра любому пользователю
- пользователь может присоединиться без приглашения
- может отображаться с учетом геолокации пользователя
- имеет ограничение по количеству участников (опционально)

---

### Персональная встреча

Персональная встреча доступна только ограниченному кругу пользователей.

Особенности:
- не отображается в общем списке встреч
- доступна только:
  - организатору
  - приглашённым участникам
- присоединение возможно только по:
  - приглашению
  - ссылке

---

## Количество гостей (для публичных встреч)

Публичная встреча может иметь ограничение на максимальное количество участников.

### Поведение

- организатор может указать максимальное количество гостей при создании встречи
- поле является необязательным
- если не задано — количество участников не ограничено

---

### Логика заполнения

- учитываются только участники, подтвердившие участие
- при достижении лимита:
  - новые пользователи не могут присоединиться
  - кнопка "Присоединиться" становится недоступной

---

### Отображение

В публичных встречах отображается:

- текущее количество участников
- максимальное количество (если задано)

Пример:
- 8 / 10 участников
- или просто 8 участников (если лимита нет)

---

### Состояния

| Состояние | Поведение |
|---|---|
| Места есть | можно присоединиться |
| Места закончились | присоединение недоступно |

---

## Поведение при создании встречи

При создании встречи пользователь должен выбрать тип:

- Персональная (по умолчанию)
- Публичная

Если выбрана публичная:
- появляется дополнительное поле:
  - **Максимальное количество гостей** (необязательно)

---

## Отображение встреч

### Раздел "Встречи"

Страница встреч должна быть разделена на два раздела:

- **Мои встречи**
- **Публичные встречи**

---

### Публичные встречи

В списке отображаются:
- название
- дата и время
- место
- краткое описание
- организатор
- количество участников (и лимит, если есть)

Дополнительно:
- пометка, что встреча публичная
- кнопка **"Присоединиться"**

---

### Персональные встречи

- отображаются только у участников
- не попадают в публичный список
- полностью сохраняют текущую логику отображения

---

## Детальная страница встречи

### Публичная встреча

- доступна для просмотра всем пользователям
- отображает:
  - список участников
  - текущее количество гостей
  - лимит (если есть)

Пользователь может:
- присоединиться (если есть свободные места)

---

### Персональная встреча

- доступна только участникам
- при отсутствии доступа пользователь не может открыть страницу

---

## Присоединение к встрече

### Публичная встреча

Пользователь может:
- нажать кнопку **"Присоединиться"**
- автоматически стать участником встречи

Ограничение:
- если достигнут лимит гостей — присоединение невозможно

---

### Персональная встреча

Пользователь может:
- присоединиться только по приглашению
- или по ссылке приглашения

---

## Геолокация (для публичных встреч)

Публичные встречи могут отображаться с учетом местоположения пользователя.

Поведение:
- при открытии списка публичных встреч приложение может учитывать геолокацию
- пользователю показываются встречи поблизости
- если геолокация недоступна — отображаются все встречи

---

## Визуальные различия

Каждая встреча должна иметь визуальное обозначение типа:

- Публичная встреча — пометка "Публичная"
- Персональная встреча — пометка "Персональная"

Дополнительно для публичных:
- отображение количества гостей

---

## Уведомления

Пользователь может получать уведомления:

- при присоединении новых участников (для организатора)
- при приглашении (для персональных встреч)
- при достижении лимита гостей (опционально)

---

## Влияние на текущую функциональность

Существующие функции остаются без изменений:

- управление участниками
- сборы средств
- опросы
- списки вещей
- карпулинг
- уведомления

Изменяется только:
- логика доступа к встречам
- способ отображения встреч
- добавляется ограничение по участникам для публичных встреч

---

## Основные пользовательские сценарии

### Найти встречу

Пользователь:
1. открывает раздел публичных встреч
2. видит количество свободных мест
3. выбирает встречу
4. присоединяется (если есть места)

---

### Создать закрытую встречу

Пользователь:
1. создает встречу
2. выбирает "Персональная"
3. приглашает участников
4. встреча доступна только им

---

### Создать открытую встречу с лимитом

Пользователь:
1. создает встречу
2. выбирает "Публичная"
3. указывает количество гостей
4. встреча появляется в общем списке
5. участники заполняют лимит

---

## Ожидаемый результат

После реализации:

- пользователи смогут находить встречи и видеть доступные места
- организаторы смогут ограничивать количество участников
- повысится управляемость публичных мероприятий
- сохранится приватность личных встреч
